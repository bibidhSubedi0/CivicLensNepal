# CivicLens Nepal: Setup & Usage

> For anyone who wants to run this locally, contribute, or just see how it works, this is everything you need.

RAG over Nepali governance documents. Retrieves relevant chunks from a local ChromaDB index and generates answers using Groq's API.

---

## Requirements

- Python 3.10+
- CUDA-capable GPU recommended (GTX 1650 or better). CPU works but embedding 67k chunks will take a very, very, VERY long time.
- Groq API key — free tier at [console.groq.com](https://console.groq.com)

---

## Installation

```bash
# clone the repo
git clone https://github.com/yourusername/CivicLensNepal.git
cd CivicLensNepal

# create and activate a virtual environment
python -m venv cln_env
cln_env\Scripts\activate        # Windows
source cln_env/bin/activate     # Linux/Mac

# install dependencies
pip install -r requirements.txt
```

### Preeti font support (optional but recommended)

A lot of older Nepali government PDFs use Preeti, a legacy ASCII-mapped font. Without this, those documents fall back to OCR which is slower and less accurate.

```bash
git clone https://github.com/sapradhan/nep-ttf2utf.git
```

The pipeline expects it at `nep-ttf2utf/` in the project root.

## Environment Variables

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here

# optional — only needed if using tesseract OCR as fallback
# easyocr is used by default if installed, tesseract is the last resort
POPPLER_PATH=C:\path\to\poppler\bin
```

## Running the Pipeline

The pipeline has three stages: **extract → chunk → embed**. Run them all at once or individually.

### Full pipeline (extract + chunk + embed)

```bash
python pipeline.py
```

### Individual stages

```bash
# extract text from PDFs only
python pipeline.py --extract-only

# chunk already-extracted text only
python pipeline.py --chunk-only

# embed already-chunked text into ChromaDB only
python pipeline.py --embed-only
```

### Flags

| Flag | Description |
|------|-------------|
| `--extract-only` | Stop after text extraction, don't chunk or embed |
| `--chunk-only` | Run chunking only (assumes extraction already done) |
| `--embed-only` | Run embedding only (assumes chunking already done) |
| `--skip-ocr` | Skip OCR fallback — only processes text-layer and Preeti PDFs. Much faster if you don't care about scanned docs. |
| `--folder FOLDER` | Process only a specific folder, e.g. `--folder 11_laws_np` |
| `--batch-size N` | Embedding batch size. Default is `64`. Use `128` for a GTX 1650, `256+` for newer cards. Higher = faster but more VRAM. |

### Examples

```bash
# embed only, with a larger batch size for a better GPU
python pipeline.py --embed-only --batch-size 256

# extract and chunk a single folder, skip OCR
python pipeline.py --extract-only --folder 07_key_laws --skip-ocr

# process just the Nepali laws folder end to end
python pipeline.py --folder 11_laws_np --batch-size 128
```

The pipeline is resumable — it skips files that have already been processed. Safe to re-run if something crashes midway.

Logs are written to `pipeline.log` in the project root.

## Running the Server

```bash
uvicorn server:app --reload --port 8000
```

Then open `http://localhost:8000` in your browser.

The API docs (auto-generated by FastAPI) are at `http://localhost:8000/docs`.

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/` | GET | Frontend chat UI |
| `/query` | POST | Submit a question, get answer + sources |
| `/health` | GET | Health check, returns chunk count and model |
| `/stats` | GET | Collection stats |
| `/broke` | GET | Quota error page |
| `/error` | GET | Generic error page |

### Query format

```json
POST /query
{
  "question": "What are the fundamental rights in Nepal's constitution?"
}
```

Response:
```json
{
  "answer": "The Constitution guarantees... [SOURCE 1][SOURCE 3]",
  "sources": [
    {
      "index": 1,
      "source": "Constitution-en.pdf",
      "category": "constitution",
      "language": "en",
      "score": 0.9123
    }
  ]
}
```

```bash
# single question

```

## Notes

- The embedding model (`intfloat/multilingual-e5-base`) is downloaded automatically on first run from HuggingFace. It's about 1.1GB.
- ChromaDB stores everything locally in `data/chromadb/`. No external database needed.
- The pipeline uses `intfloat/multilingual-e5-base` which handles both Nepali and English well. Queries need the `query:` prefix and passages need `passage:` — this is handled automatically, don't change it or retrieval quality drops noticeably.
- If you run out of Groq free tier quota, the server redirects to `/broke` automatically. Get a new key at [console.groq.com](https://console.groq.com) and update `.env`.
